LeetCode Summary 1 ———— Hash Table

Posted on 2017-03-22 | In Algorithm and Data Structrues

在 C++ STL (C++ 11) 中，哈希表这一项数据结构对应的容器是 unordered_map，定义于头文件 \。

Key Points

哈希表使用键值对 Key -value，即使用关键值 (Key) 来存储和访问数据(Value)。而键值对的对应关系通常是采用一个散列函数 (Hash function)来把关键值映射到表中的一个位置来记录访问。所以使用哈希表会增加额外内存的使用，但查找的平均时间为 O(1) 。
在哈希表中，一个重要的问题就是解决冲突的发生，即一个新的元素的插入可能会和原来已经存在的某个元素有相同的散列值。解决冲突的方法主要有分离链接法 (Separate chaining) 和开放定址法 (Open addressing)：

  • 分离链接法：将散列到同一个值的元素保留到一个链接表中。先使用散列函数确定查找哪个链接表，再在确定的链接表中执行一次查找。
  • 开放定址法：这种方法不使用额外的数据结构，而是通过一个冲突解决函数 (如线性探测，平方探测，双散列)，去尝试另外一些单元，直到找出空的单元。

2. Substring with Concatenation of All words(Description)

在 C++ 的 unordered_map 中，如果 map[word] 并不存在，则使用 map[word]++ 会直接在 map 中插入这一项并将其中的 int 值加1。改题使用两个 unordered_map 来确保是否一一对应，同时也消除同一个单词出现多次的情况。

3. Contiguous Array(Description)

这道题首先把为 0 的数换成了 -1， 然后很巧妙地运用哈希表来存储 sum 的值和第一次出现某个 sum 值的位置， 因为 sum 的值相同也就意味着这之间的等价 0 1 序列的0 和 1 的值相等，也就可以一遍遍历求出结果。

Summary

哈希表存储的是键值对，当题目中能明显看出需要使用一一对应的关系时（通常用在string 或者 char 和 int 的对应关系里）使用这种数据结构辅助解题就会事半功倍。
