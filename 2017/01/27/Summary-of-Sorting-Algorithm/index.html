Summary of Sorting Algorithm

Posted on 2017-01-27 | In Algorithm and Data Structrues

各种各样的排序算法应该算是算法的基本功了，这篇博客主要是我我参考网上的资料和数据结构与算法分析这本书，再加之一些自己的理解整理而成：

  • O(N²)简单排序：冒泡排序，选择排序和插入排序
  • 高效排序算法：希尔排序，堆排序，归并排序和快速排序
  • 线性时间排序：桶排序，计数排序和基数排序

O(N²)简单排序

冒泡排序 (bubble sort)

冒泡排序是最简单的排序算法之一，其思想是通过与相邻元素的比较和交换来把最小(最大)的数交换到最前面(最后面)。这个过程类似于水泡上升一样，因此得名冒泡算法。代码如下：

插入排序 (insertion sort)

插入排序是最简单的算法之一，它的思想是通过找到合适的位置插入元素来达到排序的目的。插入排序由 N-1 趟排序组成。对于 p = 1 到 N - 1 趟，插入排序保证从位置0到位置 p 上的元素为已排序状态。算法中实现元素移动而没有明显的使用交换。位置上的元素储存于 tmp，而在位置 P 之前的所有更大的元素都被向右移动一个位置，然后 tmp 被置于正确的位置上。如果一组输入数据几乎被排序，那么插入排序将运行的很快。代码如下：

O(N²)简单排序总结

对于冒泡排序，选择排序，插入排序以及其他一些隐含地执行相邻元素交换的排序而言，其时间下界为 Ω(N²)，其简单证明如下：
N个互异数的数组的平均逆序数是 N(N-1)/4 。对于一个未排序的数组，交换两个不按顺序的相邻元素恰好消除一个逆序，由于算法中还有 O(N) 量的其他工作，因此排序的运行时间为 O(N+N(N-1)/4) = O(N²)。所以，通过交换相邻元素进行排序的任何算法平均都需要 Ω(N²) 时间。
这个下界告诉我们，为了使一个排序算法以亚二次(subquadratic)或 O(N²) 的时间运行，必须执行一些比较，特别是要对相距较远的元素进行交换。一个排序算法通过删除逆序得以向前进行，而为了有效的进行，它必须每次交换删除不止一个逆序。

高效排序算法

希尔排序(Shellsort)

希尔排序又叫缩减增量排序(diminishing increment sort)，是最早一批冲破二次时间屏障的算法之一。它通过比较相距一定间隔的元素来工作，各趟比较所用的距离随着算法的的进行而减小，直到只比较相邻元素的最后一趟排序为止。
希尔排序使用一个序列 h1，h2，h3 … hk，叫做增量序列。只要 h1 = 1，任何增量序列都是可行的。在使用增量 hk 的一趟排序之后，对于每一个 i 我们都有 a[i] <= a[i+hk]，所有相隔 hk 的元素都被排序。希尔排序的重要性质是，一个hk排序的文件保持他的 hk 排序性。

###堆排序(HeapSort)
N 个元素建立二叉堆会花费 O(N) 时间，然后执行 N 次 DeleteMin 操作。按照顺序，最小的元素离开堆。通过将这些元素记录到第二个数组再拷贝回来，就会得到 N 个元素的排序。由于每个 deleteMin 花费时间 O(logN)，因此总的运行时间为 O(NlogN)。
我们可以对此想法做一些微小的improvement:

  • 首先，不使用第二个数组的额外内存空间，因为在每次deleteMin之后，堆会缩小1。因此位于堆中最后的元素就可以用来存放刚刚删去的元素。
  • 其次，按以上策略，最后形成的堆会是递减顺序。若要以更常见的递增顺序排序，那么可以在构建堆的时候构建一个大顶堆(父亲的关键字大于儿子的关键字)。
    因此，算法思路为，第一步以线性时间构建一个堆。然后通过每次将堆中最后元素与第一个元素交换，执行 N-1 次 deleteMin操作，每次将堆的大小缩减1并进行下滤。代码如下：

归并排序(MergeSort)

归并排序使用了分治 (divide-and-conquer) 策略，，以 O(NlogN)最坏时间情形运行，是递归算法的一个好的实例。
这个算法的基本操作是合并两个已排序的表。因为这两个表是已经排序的，所以若将输出放到第三个表中，则该算法可以通过对数据的一趟排序来完成。

以上的 merge 例程的设计，考虑到每个对 merge 的递归调用都将局部声明一个临时数组，那么同一时刻可能存在 logN 个临时数组造成内存浪费。一个好的方法是因为 merge 在mergeSort 的最后一行，所以这个临时数组可以声明在 mergeSort 驱动程序中，而且还可以使用这个临时数组的任意部分来排序arr数组中相同的部分。
虽然归并排序的运行时间是 O(NlogN)，但明显的问题是合并两个已排序的表要用到线性附加内存，而且整个算法中还要花费将数据拷贝到临时数组再拷贝回来的附加操作，这明显减慢了排序的速度。

快速排序(QuickSort)

快速排序是实践中的一种快速的排序方法，它的平均运行时间为 O(NlogN)。该算法之所以特别快，主要是由于非常精炼和高度优化的内部循环。其基本思想与归并排序类似，都是采用分治的思想。其基本步骤如下：

  1. 先从数列中取出一个数作为基准数。
  2. 分区过程，将比这个基准数大的数全放到它的右边，小于或等于它的数全放到它的左边。
  3. 再对左右区间重复第二步，直到各区间只有一个数。

对于快速排序而言，枢纽元的选取是非常重要的。如果输入是随机的，那么选取第一个数是可以接受的。但如果输入是预排序的，那么将花费二次时间界，而且快速排序实际上什么事也没有干。一般做法是选取第一个数，中间数，和最后一个数的中位数来作为枢纽元，实现上述方法只要将中位数与第一个数交换位置就可以了。
由快速排序引申出来的快速选择可以以 O(N) 平均时间复杂度解决选择问题：N 个元素中找到第 k 个最大元或最小元。其思想就是在递归枢纽元前和后的部分时，如果 K <= pivot, 则目标元素就在前半部分中，如果 k = pivot + 1，则目标元素就是枢纽元，否则，目标元素就在后半部分中。快速选择只做一次递归调用而不是两次，当算法终止时，目标元素就在位置 k-1 处。

高效排序算法小结

对于任何只用到比较的排序算法，在最坏情况下都需要 Ω(NlogN) 次比较，因此以上的方法在一个常数因子范围内是最优的。其简单证明如下：N个数有N!个可能的排列情况，也就是说基于比较的排序算法的决策树有N!个叶子结点，比较次数至少为 log(N!)=O(NlogN)。

线性时间排序

虽然已经证明过任何只用到比较的排序算法都无法突破 O(NlogN) 的时间界，但是在一定特殊情况下，仍然可能以线性时间来进行排序。前提条件就是待排序的数是在一定范围内整数。以下就是三种非基于比较的线性时间排序算法，计数排序，桶排序和基数排序。

桶排序(BucketSort)

桶排序把一组长度为 N 的关键字序列按照某种映射函数划分成 M 个子区间(桶)。然后基于这种映射函数，将待排序的序列映射到相对应的桶中，再对每个桶中的元素进行比较排序(如快排)。最后，按着桶的顺序依次输出每个桶的元素，得到的就是有序序列。对于桶排序而言，映射函数必须满足条件：如果如果关键字 k1 < k2，那么f(k1)<=f(k2)，也就是说B(i)中的最小数据都要大于B(i-1)中最大数据。

计数排序(CountingSort)

计数排序需要用到较多的辅助空间，其基本思想是用待排序的数作为计数数组的下标，统计每个数字的个数，然后依次输出即可得到有序序列。
算法的步骤如下：

  1. 找出待排序的数组中最大和最小的元素；
  2. 扫描序列 arr， 以 arr 中的每个元素的值为索引，把出现的个数填入 C 中，此时 C[i] 可以表示 arr 中值为 i 的元素的个数；
  3. 对所有的计数累加 (从 C 中的第一个元素开始，每一项和前一项相加)；
  4. 反向填充目标数组 B，将每个元素 i 放在新数组的 C(i) 项，每放一个元素就将 C(i) 减一 (反向填充是为了保持其稳定性，后放进去的元素相对而言先取出来)。

基数排序(RadixSort)

基数排序是一种借助多关键字排序思想对单逻辑关键词进行排序的方法.多关键字的意思是多个优先级不同的关键字，比如说对数字排位，就是十位，百位，千位等等。以上介绍的两种排序方法，桶排序和计数排序都可以作为基数排序的内部方法来实现单关键字排序，前提是要保持稳定，也就是在第K趟之后，前K-1趟排序也保持有序，这样在所有的排序之后，元素也会最终有序。

总结

排序算法有的编码简单，有的时间快速，而有的通过空间的牺牲来换取时间的高效。但是排序算法的选取并不是一成不变的，不是快速排序就一定比插入排序耗时短，在具体的情况下需要特殊分析，选取最合适的排序算法。

排序方法 平均时间 最坏情况 辅助存储
简单排序 O(N²) O(N²) O(1)
快速排序 O(NlogN) O(N²) O(logN)
堆排序 O(NlogN) O(NlogN) O(1)
归并排序 O(N²) O(NlogN) O(N)
基数排序 O(d(N+rd)) O(d(N+rd)) O(rd)
  1. 从平均时间来看，快速排序是效率最高的，但快速排序在最坏情况下的时间性能不如堆排序和归并排序。堆排序和归并排序相比，在 N 较大时归并排序时间较短，但使用辅助空间较多。
  2. 简单排序包括冒泡排序，插入排序和选择排序，其中插入排序最简单。在序列基本有序或者 N 较小时，插入排序是不错的排序方法，所以经常将它和其他的排序方法，如快排一起使用。
  3. 基数排序的时间复杂度也可以简化为O(d*N)，因此它使用与元素数量 N 很多，而关键字较小的序列。若关键字数量也很大，而序列中大多数记录的最高关键字均不同，则可以按最高关键字不同划分为若干个小的序列，再对小的序列进行插入排序。
  4. 从方法的稳定性来比较，基数排序是稳定的内排方法。所有时间复杂度为O(N²)的简单排序也是稳定的。但是快速排序、堆排序、希尔排序等时间性能较好的排序方法都是不稳定的。稳定性需要根据具体需求选择。
